# Section 6: AI Agent Frameworks

## 1. Tại Sao Cần Framework?

### Vấn Đề Khi Tự Xây Dựng Từ Đầu
- Boilerplate code phức tạp
- Error-prone operations (tool calling, parsing)
- State management khó khăn
- RAG pipelines phức tạp
- Khó maintain và scale

### Các Framework Cung Cấp
- **Tool Calling**: Format JSON schema, parse function calls
- **State Management**: Persist chat history, manage context
- **RAG Integration**: Connect với vector databases
- **Memory**: Long-term và short-term memory handling

## 2. Ba Loại Framework Chính

### A. Generalist Toolkit (Workshop)

**Đặc điểm:**
- Low-level building blocks
- Linh hoạt, tự do xây dựng mọi thứ
- Cần assembly nhiều thành phần

**Đại diện: LangChain & LangGraph**

#### LangChain
- **Mục đích**: Framework tổng quát cho LLM applications
- **Components**:
  - Prompt templates
  - Model interfaces (OpenAI, Anthropic, local models)
  - Document loaders (PDF, web, databases)
  - Vector stores (Pinecone, Chroma, Weaviate)
  - Chains (sequence of calls)
  - Agents (tool-using agents)
- **Điểm mạnh**: Linh hoạt, ecosystem lớn, nhiều integrations
- **Phù hợp**: Prototyping, custom workflows

#### LangGraph
- **Mục đích**: Xây dựng multi-agent applications với graph structure
- **Đặc điểm**:
  - State machine-based
  - Nodes là agents/functions
  - Edges define transitions
  - Cycles cho phép loops
- **Điểm mạnh**: Explicit control flow, dễ debug complex workflows
- **Phù hợp**: Multi-step, conditional workflows

**Ví dụ LangGraph:**
```python
# Define workflow graph
graph = StateGraph(AgentState)

# Add nodes (agents)
graph.add_node("router", router_agent)
graph.add_node("researcher", research_agent)
graph.add_node("writer", writing_agent)

# Add edges (transitions)
graph.add_edge("router", "researcher", condition=needs_research)
graph.add_edge("researcher", "writer")
graph.add_edge("writer", END)
```

### B. Specialist Frameworks

**Đặc điểm:**
- High-end appliances cho use cases cụ thể
- Optimized cho domain riêng

**Đại diện 1: LlamaIndex**

#### LlamaIndex
- **Mục đích**: Framework chuyên về **data ingestion và RAG**
- **Đặc điểm**:
  - Data connectors (300+ sources: SQL, PDF, Notion, Slack...)
  - Indexing strategies (vector, tree, keyword)
  - Query engines (retrieve → synthesize)
  - Agentic RAG (agent có thể query nhiều sources)
- **Điểm mạnh**: 
  - Data retrieval xuất sắc
  - Complex query handling
  - Structured data extraction
- **Phù hợp**: 
  - Chat with documents
  - Knowledge base Q&A
  - Enterprise data search

**Ví dụ LlamaIndex:**
```python
# Load documents
documents = SimpleDirectoryReader("data").load_data()

# Create index
index = VectorStoreIndex.from_documents(documents)

# Query
query_engine = index.as_query_engine()
response = query_engine.query("What are the key findings?")
```

**Đại diện 2: Microsoft Agent Framework**

#### Microsoft Agent Framework (AutoGen)
- **Mục đích**: Chuyên về **multi-agent collaboration**
- **Đặc điểm**:
  - Conversational agents
  - Group chat between agents
  - Human proxy (HITL)
  - Code execution agents
- **Điểm mạnh**:
  - Agent conversation xuất sắc
  - Code generation and execution
  - Flexible agent patterns
- **Phù hợp**:
  - Multi-agent problem solving
  - Coding assistants
  - Collaborative research

### C. Managed Platforms (Cloud Agents)

**Đặc điểm:**
- Full-service factory
- Mua agent-as-a-service
- Không cần code nhiều

**Đại diện:**
- **AWS Bedrock Agents**
- **Azure AI Agent Service**
- **Google Vertex AI Agents**

**Lợi ích:**
- Managed infrastructure
- Built-in security
- Enterprise integration
- Monitoring and logging

**Trade-off:**
- Less customization
- Vendor lock-in
- Cost subscription

## 3. Lựa Chọn Framework Theo Use Case

| Use Case | Recommended Framework |
|----------|---------------------|
| Chat with documents | LlamaIndex |
| Multi-agent coding | AutoGen (Microsoft) |
| Complex workflow logic | LangGraph |
| Prototyping, flexibility | LangChain |
| Enterprise managed solution | Azure/AWS/GCP Agents |
| Visual workflow building | Flowise, n8n |

## 4. Trade-off Summary

| Approach | Flexibility | Complexity | Maintenance | Best For |
|----------|------------|------------|-------------|----------|
| Raw API | High | Very High | Very Hard | Learning |
| LangChain | High | Medium | Medium | Custom apps |
| LangGraph | Medium | Medium | Medium | Complex flows |
| LlamaIndex | Medium (data) | Low | Low | RAG apps |
| AutoGen | Medium | Medium | Medium | Multi-agent |
| Cloud Platforms | Low | Low | Very Low | Enterprise |
