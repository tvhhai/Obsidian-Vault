# Section 5: Agentic Workflows

## 1. From Traditional to Agentic Workflows

### Traditional Workflow
```
Fixed Steps: A → B → C → D
- Deterministic (tất định)
- Hard-coded logic
- No adaptation (không thích ứng)
```

### Agentic Workflow
```
Dynamic Path: Start → [Agent decides] → [Action/Tool] → [Observe] → [Decide next]
- Non-deterministic (phi tất định)
- LLM-driven decisions
- Adaptive to context (thích ứng với ngữ cảnh)
```

## 2. Components of Agentic Workflow

### State (Trạng Thái)
- Dữ liệu được truyền qua workflow
- **Chứa**:
  - Input của user
  - Intermediate results (kết quả trung gian)
  - Memory từ các bước trước
- **Ví dụ**: `AgentState` chứa messages, current_step, accumulated_data

### Nodes (Nút)
- Các đơn vị xử lý trong workflow
- **Mỗi node là**:
  - Một agent (LLM call)
  - Một function (Python logic)
  - Một tool call

### Edges (Cạnh)
- Kết nối giữa các nodes
- **Định nghĩa flow**:
  - **Simple edge**: A → B (luôn chuyển từ A sang B)
  - **Conditional edge**: A → [condition] → B hoặc C

## 3. Agentic Orchestration Patterns

### A. Sequential Pattern
```
Node 1 → Node 2 → Node 3 → End
```
- Các bước thực hiện tuần tự
- Output của bước trước là input của bước sau

**Ví dụ**: Content pipeline (Research → Write → Edit)

---

### B. Concurrent Pattern (Parallel)
```
        ┌→ Node 1 →┐
Start → ├→ Node 2 →├ → Aggregator → End
        └→ Node 3 →┘
```
- Nhiều nodes chạy song song
- Kết quả được aggregate (tổng hợp)

**Ví dụ**: Parallel research từ nhiều sources

---

### C. Group Chat Pattern
```
   ┌─────┐
   │Agent A│←────→
   └──┬──┘      │
      ↑    Group Chat    ↓
   ┌──┴──┐    Manager ┌──┴──┐
   │Agent B│←────────→│Agent C│
   └─────┘            └─────┘
```
- Nhiều agents thảo luận với nhau
- Group chat manager điều phối

---

### D. Handoff Pattern
```
User → Agent A (Triage) → Agent B (Process) → Agent C (Resolve) → End
                              ↓ (nếu cần)
                         Human Agent
```
- Task được chuyển tay giữa các agents
- Mỗi agent xử lý phần việc rồi chuyển tiếp

## 4. Frameworks for Building Workflows (Code-First)

### LangGraph
```python
from langgraph.graph import StateGraph, END

# Define state
class AgentState(TypedDict):
    messages: list
    next_agent: str

# Build graph
workflow = StateGraph(AgentState)

# Add nodes
workflow.add_node("agent_a", agent_a_func)
workflow.add_node("agent_b", agent_b_func)

# Add edges
workflow.add_conditional_edges(
    "agent_a",
    route_decision,
    {"to_b": "agent_b", "end": END}
)

# Compile
app = workflow.compile()
```

### LlamaIndex Workflows
```python
from llama_index.core.workflow import Workflow, Event

class MyWorkflow(Workflow):
    @step
    async def step_one(self, ev: StartEvent) -> ProcessEvent:
        # Processing
        return ProcessEvent(result=result)
    
    @step
    async def step_two(self, ev: ProcessEvent) -> StopEvent:
        # Final processing
        return StopEvent(result=final_result)
```

## 5. Platforms for Building Workflows (Low-Code)

### Đặc điểm
- Visual workflow builders
- Drag-and-drop interface
- Minimal code required

### Các Platform
- **n8n**: Open-source workflow automation
- **Make (Integromat)**: Visual integration platform
- **Zapier with AI**: Workflow automation with AI steps
- **Flowise**: Visual LangChain builder
